Apollo基础配置

官方文档：Apollo (apolloconfig.com)

windows 本地服务端测试环境快速搭建

确保本地有对应版本的jdk环境和mysql环境，和git bash环境方便在windos上执行shell脚本

按照官方文档选择需要的apollo版本直接下载解压到本地，将sql文件夹中的sql在数据库先执行完毕，会创建两个数据库，分别为ApolloConfigDB，ApolloPortalDB，打开编辑damo.sh 脚本，修改脚本文件中的数据库连接配置，然后再根目录打开git bash 执行demo.sh 脚本即可

docker quick-start搭建服务端

准备docker环境

准备好docker-compose.yml 容器启动配置文件，与建表sql文件都可以去官网下载

在linux本机或者宿主机上 找一个位置创建docker-quick-start目录(名字随意)，并按照如下目录结构配置好

–docker-quick-start

​ –docker-compose.yml

​ –sql

​ –apolloconfigdb.sql

​ –apolloportaldb.sql

然后直接运行 docker-compose up命令

如果报错找不到命令 直接yum install docker-compose -y安装后重新运行即可

客户端搭建

本地springboot项目直接maven引入

配置文件中增加配置

项目启动类添加注解@EnableApolloConfig即可

apollo支持@Value和Environment动态刷新，而对于@ConfigurationProperties，能加载到配置，但是无法自动刷新
