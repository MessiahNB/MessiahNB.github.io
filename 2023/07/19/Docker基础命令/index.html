Docker 基本操作

常用命令

  • sndo systemctl start docker 启动docker
  • sudo systemctl status docker 查看docker状态
  • sudo systemctl restart docker 重启docker
  • sudo systemctl stop docker 停止docker
  • docker version 查看docker的版本号
  • docker pull 应用镜像名:版本号 在镜像仓库下载镜像
  • docker images 查出当前docker所有镜像（包含下载镜像与自己创建的镜像）
  • docker rmi 镜像tag/id 删除镜像 这里面还有说法。。。 docker rmi -f 强制删除、

批量删除镜像的两种方式

  1. 使用$() 表达式进行删除

  2. 使用xargs

  • docker save -o 导出的tar文件名 仓库:标签名 导出镜像
  • docker load -i xx.tar 导入镜像
  • docker image prune 清理镜像

容器命令

  • docker ps 列出正在运行的Docker容器

  • docker run 镜像tag 启动镜像创建容器，在docker中容器的启动方式有两种 一种是：交互式启动 一种是：后台启动

  • docker exec -it 容器id/容器名称 /bin/bash 进入一个正在运行的容器

    进入容器后退出容器 有两种方式 1 直接输入 exit 退出容器并停止容器 2 快捷键 ctrl + p +q 退出容器但不停止容器运行

  • docker start 容器名称或容器ID 启动一个已停止的容器

  • docker stop 容器名称或容器ID 停止容器

  • docker restart 容器名称或容器ID 重新启动一个已停止的容器

  • docker rm -f 容器名称或容器id 删除容器，其中-f为强制删除

  • 导出容器

  • 导入容器

  • docker top 容器id或容器名称 查看容器进程

  • docker stats 容器id或容器名称 查看容器内CPU、内存、存储或网络信息

  • docker inspect 容器id或容器名称 查看容器详情

  • docker cp 容器id或容器名:容器中文件路径 主机路径 复制容器文件到主机

  • docker cp 主机文件路径 容器id或容器名:容器中文件路径 拷贝主机的文件到容器中

  • docker diff 容器id或容器名称 diff 看容器内文件系统的变更 类似与操作日志

  • docker port 容器id或容器名称 查看容器占用端口信息

  • docker logs --tail 10 容器id或容器名称 查看容器最后十条日志记录

  • docker build -t 镜像名称:tag . 将当前目录作为（软件包）构建上下文，并根据dockerfile构建生成一个镜像文件

Docker容器管理

docker run 等于创建+启动

docker run 镜像名 如果镜像本地不存在，则会在线去下载该镜像

注意：容器内的进程必须处于前台运行状态，否则容器就会直接退出

自己部署容器运行的时候，命令不得后台运行，前台运行即可

如果容器内什么事儿也没有做，那么容器也会挂掉，容器内必须有一个进程在前台运行

像运行nginx的基础镜像，没有运行任何程序因此容器直接挂掉

Dockerfile

要想获得一个新的镜像除了 通过docker commit 将一个容器提交为一个新的镜像这种方式外，另外就是通过dockerfile了

官方提供的dockerfile示例

https://github.com/Centos/Centos-Dockerfiles

dockerfile 的主要组成部分

基础镜像信息 FROM centos:6.8

制作镜像操作指令 RUN yum install openssh-server -y

容器启动时执行指令 CMD [“/bin/bash”]

dockerfile指令

FROM 这个镜像的妈妈是谁（指定基础镜像）

MAINTAINER 告诉别人谁负责养它？（指定维护信息，可以没有）

RUN 你想让它干啥 ( 在命令前面加上RUN即可)

ADD 给它点创业资金，将宿主机的文件添加到容器内（COPY文件，会自动解压）

COPY 和ADD是一样的作用 只不过不会自动解压 仅仅只是拷贝

WORKDIR 我是cd 今天刚化了妆（设置当前工作目录）

VOLUME 给它一个存放行李的地方（设置卷，挂载主机目录）数据目录映射

EXPOSE 它要打开的门是啥（指定对外的端口）

CMD 奔跑吧兄弟 （指定容器启动后要干的事情）

COPY

copy 指令从宿主机复制文件或者目录到新的一层镜像内

如

COPY messiah.txt /home/

支持多个文件，以及通配符形式复制，语法满足Golang的filepath.Match

COPY messiah* /tem/mess?.txt /home/

并且能够保留源文件的元数据，如权限访问时间等等 这很重要

ADD

特性和copy基本一致，不过多了些功能

1 源文件是一个压缩文件，且是gzip、bzip2、xz、tar情况，add指定会自动解压缩该文件到目标路径

2 源文件是一个url，且是一个压缩包，不会自动解压还是需要 run tar -zxf 单独解压

3 源文件是一个url，docker引擎会下载该链接，放入目标路径，且权限自动设为600 若不是期望结果还是需要同run 调整

Dockerfile 官方更为推荐使用copy，add包含了更多复杂的功能，且ADD会使构建缓存失效，导致镜像构建缓慢

CMD

cmd在容器内运行某个指令，启动程序

CMD [“cat”,”/etc/os-release”] 查看系统版本

这行命令等同于 docker run -it centos7 cat /etc/os-release

注意 如果需要在容器内启动nginx

不能 RUN systemctl start nginx 也不能 CMD [“sh”,”-c”,”systemctl start nginx”]

而是 CMD [“nginx”,”-g”,”daemon off;”]

ENTRYPOINT

作用和CMD一样，都是在指定容器启动程序和参数

但是需要注意的是 在dockerfile中指定了CMD外 又指定了ENTRYPOINT,那么CMD的语义就发生了变化

会把CMD的内容当作参数传递给ENTRYPOINT

在实际使用中 CMD 后面的参数 会被 我们运行镜像时所携带的参数给覆盖 但是ENTRYPOING的话 就会在后面追加

ARG和ENV

都是设置环境变量的

区别时 env设置的环境变量无论是在构建镜像时 还是容器运行过程中都是存在的，而arg设置的变量就只会在构建镜像时生效

VOLUM

将指定文件目录挂载到容器内，运行在容器内产生的数据应该挂载到宿主机上

容器数据挂载的方式 通过dockerfile 指定VOLUM目录 VOLUM [“/data1”,”/data2”]

通过docker run -v 参数，直接设置需要映射挂载的目录

EXPOSR

指定容器运行时对外提供的端口服务

WORKDIR

用于在dockerfile脚本中进行目录的切换

WORKDIR /etc

USER

用于改变环境，切换用户
